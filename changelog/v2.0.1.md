# 2.0.1 — In Development

## Security

| Tag | Change |
|---:|---|
| ![bugfix] | <details><summary>Async forwarding rejected valid client requests on the master server</summary><br>When a client played directly on the master server, `handleOnServer` passed an empty slave ID. The untrusted-slave security check fired first and rejected the request because the empty ID wasn't in the trusted list — the client whitelist was never consulted. Extracted security logic into a shared `isRequestAllowed` method that skips the slave trust check when no slave is involved.</details> |
