## 🔴 Critical Fixes

### Security
- **Fixed** - Permission system bypass vulnerability
  - Replaced incorrect `.ordinal()` calls with `.getValue()` in permission checks
  - Affected files: DepositItemsInBankRequest, WithdrawItemsFromBankRequest, AsyncBankManager
  - [Issue #1](.claude/ISSUES.md#issue-1)

---

## 🐛 Bug Fixes

### Banking Logic
- **Fixed** - `withdrawLockedAsync()` incorrectly calling `withdraw()` instead of `withdrawLocked()`
  - [Issue #3](.claude/ISSUES.md#issue-3)
- **Fixed** - `hasUserAsync()` returning wrong result (hasAnyUser instead of hasUser)
  - [Issue #4](.claude/ISSUES.md#issue-4)
- **Fixed** - `hasBankAsync()` returning wrong result (hasAnyBank instead of hasBank)
  - [Issue #5](.claude/ISSUES.md#issue-5)
- **Fixed** - Missing return statement in `createBankAccountAsync()` causing potential NPE
  - [Issue #6](.claude/ISSUES.md#issue-6)
- **Fixed** - Integer overflow protection in balance calculations
  - [Issue #7](.claude/ISSUES.md#issue-7)
- **Fixed** - Display bug in `getNormalizedTotalBalanceAsync()`
  - [Issue #10](.claude/ISSUES.md#issue-10)
- **Fixed** - Display bug in `getFormattedTotalBalanceAsync()`
  - [Issue #11](.claude/ISSUES.md#issue-11)

### Concurrency
- **Fixed** - Potential ConcurrentModificationException in listener unsubscription
  - [Issue #8](.claude/ISSUES.md#issue-8)
- **Evaluated** - Thread safety analysis for balance operations
  - [Issue #2](.claude/ISSUES.md#issue-2)

### Networking
- **Fixed** - `removeAllBanksAsync()` using wrong function type
  - [Issue #9](.claude/ISSUES.md#issue-9)

### Code Quality
- **Improved** - Null handling in `setAccountName()`
  - [Issue #12](.claude/ISSUES.md#issue-12)
- **Fixed** - Equality check for ItemID in `setAccountIcon()`
  - [Issue #13](.claude/ISSUES.md#issue-13)
