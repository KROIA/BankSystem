---

## 🔴 Critical Fixes

### Security
- **Fixed** - Permission system bypass vulnerability
  - Replaced incorrect `.ordinal()` calls with `.getValue()` in permission checks
  - Affected files: DepositItemsInBankRequest, WithdrawItemsFromBankRequest, AsyncBankManager

---

## 🐛 Bug Fixes

### Banking Logic
- **Fixed** - `withdrawLockedAsync()` incorrectly calling `withdraw()` instead of `withdrawLocked()`
- **Fixed** - `hasUserAsync()` returning wrong result — was calling `hasAnyUser()` instead of `hasUser(userUUID)`
- **Fixed** - `hasBankAsync()` returning wrong result — was calling `hasAnyBank()` instead of `hasBank(itemID)`
- **Fixed** - Missing return statement in `createBankAccountAsync()` causing potential NPE
- **Fixed** - Integer overflow protection in balance calculations
- **Fixed** - Display bug in `getNormalizedTotalBalanceAsync()` — was using `lockedBalance` instead of `getTotalBalance()`
- **Fixed** - Display bug in `getFormattedTotalBalanceAsync()` — was using `lockedBalance` instead of `getTotalBalance()`

### Concurrency
- **Fixed** - Potential ConcurrentModificationException in listener unsubscription
- **Evaluated** - Thread safety analysis for balance operations

### Networking
- **Fixed** - `removeAllBanksAsync()` using wrong function type

### Code Quality
- **Improved** - Null handling in `setAccountName()`
- **Fixed** - Equality check for ItemID in `setAccountIcon()`
